# Project Onboarding Prompt

Используй этот файл как короткий стартовый prompt для нового чата.

## Рекомендуемая фраза

```text
Прочитай skills/project-onboarding.md, затем AGENTS.md и docs/onboarding/START-HERE.md. После чтения кратко опиши текущее состояние проекта, архитектурные ограничения, mobile-first приоритет и что уже реализовано в коде.
```

## Зачем это нужно

Этот файл нужен, чтобы не вспоминать каждый раз вручную, какие документы давать агенту первыми.

## Что читать дальше

После этого файла агент должен открыть:

1. [AGENTS.md](/g:/vs/auto pets/AGENTS.md:1)
2. [docs/onboarding/START-HERE.md](/g:/vs/auto pets/docs/onboarding/START-HERE.md:1)

А внутри `START-HERE.md` уже есть полный порядок чтения.

## Обновление этого файла

Если в проекте появляются новые важные skills, новые onboarding-документы, меняется рекомендуемый порядок чтения, или меняется platform priority проекта, агент должен обновить этот файл в той же задаче.

Обновлять нужно только устойчивые изменения, которые действительно помогут следующему пустому чату быстрее войти в курс дела.

Особенно важно обновлять этот файл, если:

- добавлен новый skill в `skills/`
- появился новый ключевой документ в `docs/`
- изменился главный стартовый prompt для нового чата
- появились новые обязательные файлы для чтения
- стало критично, что проект делается `mobile-first`
